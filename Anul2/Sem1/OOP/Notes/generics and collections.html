Java Collections Framework (JCF)

JCF is a unified framework for managing collections of objects. It includes:

  1. Interfaces: Abstract data types like List, Set, Map.
  2. Implementations: Concrete classes such as ArrayList, HashSet, HashMap.
  3. Utility Methods: Algorithms for operations like sorting, searching, and shuffling.

Why Use JCF?

  • Simplifies development with pre-built data structures and algorithms.
  • Provides type safety and interoperability.
  • Enhances performance and code maintainability.

Key Collections and Their Examples

1. List

A sequence of elements that allows duplicates and preserves insertion order.

  • Common Implementations:

    • ArrayList: Dynamic array, fast random access.
    • LinkedList: Doubly linked list, efficient for frequent insertions/removals.

  • Examples:

    // Using ArrayList
List<String> arrayList = new ArrayList<>();
arrayList.add("Apple");
arrayList.add("Banana");
arrayList.add("Apple"); // Duplicates allowed
System.out.println(arrayList); // Output: [Apple, Banana, Apple]

// Using LinkedList
List<Integer> linkedList = new LinkedList<>();
linkedList.add(1);
linkedList.add(2);
linkedList.add(3);
linkedList.remove(1); // Removes element at index 1
System.out.println(linkedList); // Output: [1, 3]

2. Set

A collection of unique elements, with no duplicate values.

  • Common Implementations:

    • HashSet: Uses a hash table for fast lookups.
    • TreeSet: Maintains sorted order using a binary tree.

  • Examples:

    // Using HashSet
Set<String> hashSet = new HashSet<>();
hashSet.add("Dog");
hashSet.add("Cat");
hashSet.add("Dog"); // Duplicate ignored
System.out.println(hashSet); // Output: [Dog, Cat]

// Using TreeSet
Set<Integer> treeSet = new TreeSet<>();
treeSet.add(10);
treeSet.add(5);
treeSet.add(20);
System.out.println(treeSet); // Output: [5, 10, 20]

3. Map

A collection of key-value pairs, where keys are unique.

  • Common Implementations:

    • HashMap: Unordered, fast lookups.
    • TreeMap: Sorted by key.
    • LinkedHashMap: Maintains insertion order.

  • Examples:

    // Using HashMap
Map<String, Integer> hashMap = new HashMap<>();
hashMap.put("Alice", 25);
hashMap.put("Bob", 30);
hashMap.put("Alice", 28); // Overwrites value for "Alice"
System.out.println(hashMap); // Output: {Alice=28, Bob=30}

// Using TreeMap
Map<Integer, String> treeMap = new TreeMap<>();
treeMap.put(3, "Three");
treeMap.put(1, "One");
treeMap.put(2, "Two");
System.out.println(treeMap); // Output: {1=One, 2=Two, 3=Three}

4. Queue and Deque

Used for element retrieval based on FIFO or LIFO order.

  • Common Implementations:

    • PriorityQueue: Elements are ordered by natural ordering or a comparator.
    • ArrayDeque: Double-ended queue.

  • Examples:

    // Using PriorityQueue
Queue<Integer> priorityQueue = new PriorityQueue<>();
priorityQueue.add(20);
priorityQueue.add(10);
priorityQueue.add(30);
System.out.println(priorityQueue.poll()); // Output: 10 (smallest element)

// Using ArrayDeque
Deque<String> deque = new ArrayDeque<>();
deque.addFirst("First");
deque.addLast("Last");
System.out.println(deque.pollFirst()); // Output: First
System.out.println(deque.pollLast());  // Output: Last

Utility Methods in Collections

Java provides utility methods via the Collections class for operations on collections:

  1. Sorting:

    List<String> list = new ArrayList<>(Arrays.asList("Zebra", "Monkey", "Apple"));
Collections.sort(list);
System.out.println(list); // Output: [Apple, Monkey, Zebra]

  2. Shuffling:

    Collections.shuffle(list);
System.out.println(list); // Output: Random order

  3. Finding Min/Max:

    int max = Collections.max(Arrays.asList(1, 2, 3, 4));
System.out.println(max); // Output: 4

Overview of Generics

Generics allow you to create classes, interfaces, and methods that operate on types specified at runtime, ensuring type safety and flexibility.

Syntax Overview

  • T, E, K, V are common placeholders for types:
    • T – Type.
    • E – Element (used in collections).
    • K, V – Key and Value (used in maps).
  • ? – Wildcard for unknown types.

Generic Classes

A generic class uses type parameters to ensure type safety.

  • Example: Generic Container Class

    public class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

// Usage
Box<Integer> intBox = new Box<>();
intBox.set(123);
System.out.println(intBox.get()); // Output: 123

Box<String> strBox = new Box<>();
strBox.set("Hello Generics!");
System.out.println(strBox.get()); // Output: Hello Generics!

Generic Methods

A method can define its own type parameter, independent of the class.

  • Example: Method to Find Maximum of Two Elements

    public static <T extends Comparable<T>> T findMax(T a, T b) {
    return a.compareTo(b) > 0 ? a : b;
}

// Usage
System.out.println(findMax(3, 7)); // Output: 7
System.out.println(findMax("Apple", "Banana")); // Output: Banana

Bounded Type Parameters

You can restrict a generic type to a specific class or interface using extends (upper bound) or super (lower bound).

  • Example: Upper-Bounded Generics

    public static <T extends Number> double calculateSum(T a, T b) {
    return a.doubleValue() + b.doubleValue();
}

// Usage
System.out.println(calculateSum(5, 7)); // Output: 12.0
System.out.println(calculateSum(5.5, 7.3)); // Output: 12.8

  • Example: Lower-Bounded Generics

    public static void addNumberToList(List<? super Integer> list) {
    list.add(10);
}

// Usage
List<Number> numbers = new ArrayList<>();
addNumberToList(numbers);
System.out.println(numbers); // Output: [10]

Wildcard Types

Wildcards (?) are used when the exact type is unknown.

  • Unbounded Wildcard (?) Allows any type.

    public static void printList(List<?> list) {
    for (Object elem : list) {
        System.out.println(elem);
    }
}

// Usage
List<String> strings = List.of("A", "B", "C");
printList(strings); // Output: A B C

  • Upper-Bounded Wildcard (? extends T) Limits to T and its subclasses.

    public static double sumNumbers(List<? extends Number> list) {
    double sum = 0.0;
    for (Number n : list) {
        sum += n.doubleValue();
    }
    return sum;
}

// Usage
List<Integer> integers = List.of(1, 2, 3);
List<Double> doubles = List.of(1.1, 2.2, 3.3);
System.out.println(sumNumbers(integers)); // Output: 6.0
System.out.println(sumNumbers(doubles)); // Output: 6.6

  • Lower-Bounded Wildcard (? super T) Accepts T and its supertypes.

    public static void addElements(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}

// Usage
List<Number> numbers = new ArrayList<>();
addElements(numbers);
System.out.println(numbers); // Output: [1, 2, 3]

Generic Interfaces

Generic interfaces allow flexible implementations.

  • Example: Generic Interface for a Pair

    public interface Pair<K, V> {
    K getKey();
    V getValue();
}

public class KeyValuePair<K, V> implements Pair<K, V> {
    private K key;
    private V value;

    public KeyValuePair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}

// Usage
Pair<Integer, String> pair = new KeyValuePair<>(1, "One");
System.out.println(pair.getKey()); // Output: 1
System.out.println(pair.getValue()); // Output: One

Real-World Examples

1. Filtering Elements in a Collection

Use generics to filter elements of a specific type.

public static <T> List<T> filter(List<T> list, Predicate<T> predicate) {
    List<T> result = new ArrayList<>();
    for (T element : list) {
        if (predicate.test(element)) {
            result.add(element);
        }
    }
    return result;
}

// Usage
List<String> strings = List.of("Apple", "Banana", "Cherry");
List<String> filtered = filter(strings, s -> s.startsWith("B"));
System.out.println(filtered); // Output: [Banana]

2. Swapping Elements in an Array

public static <T> void swap(T[] array, int i, int j) {
    T temp = array[i];
    array[i] = array[j];
    array[j] = temp;
}

// Usage
Integer[] numbers = {1, 2, 3};
swap(numbers, 0, 2);
System.out.println(Arrays.toString(numbers)); // Output: [3, 2, 1]

3. Generic Stack Implementation

public class Stack<E> {
    private List<E> elements = new ArrayList<>();

    public void push(E element) {
        elements.add(element);
    }

    public E pop() {
        if (elements.isEmpty()) {
            throw new EmptyStackException();
        }
        return elements.remove(elements.size() - 1);
    }

    public boolean isEmpty() {
        return elements.isEmpty();
    }
}

// Usage
Stack<String> stack = new Stack<>();
stack.push("A");
stack.push("B");
System.out.println(stack.pop()); // Output: B
